What is a String?

A String is a sequence of characters representing text data. In C++, strings can be represented as character arrays (C-style) or as std::string objects with rich functionality and automatic memory management.

Visual Representation of "HELLO":

[1]

// C-style string (null-terminated) char str1[] = "Hello"; // C++ std::string std::string str2 = "Hello"; // Access characters char ch = str2[0]; // 'H' char last = str2[str2.length() - 1]; // 'O'

Key Characteristics:

  • Sequence of characters - ordered collection
  • Zero-based indexing - first character at index 0
  • Null-terminated (C-style) - ends with '\0'
  • Mutable in C++ - can modify characters
  • Dynamic sizing (std::string) - grows automatically
  • Rich standard library - extensive string operations

Types of Strings

1. C-Style String (Character Array)

Null-terminated array of characters with manual memory management.

char str[] = "Hello"; char* ptr = "World"; // Pointer to string literal

Features: Fixed size, manual null termination, pointer arithmetic

2. C++ std::string

Modern C++ string class with automatic memory management and rich API.

#include <string> std::string str = "Hello"; str += " World"; // Easy concatenation

Features: Dynamic sizing, operator overloading, bounds checking, automatic cleanup

3. std::string_view (C++17)

Non-owning, read-only reference to a string. Zero-copy string passing.

#include <string_view> std::string_view sv = "Hello"; // No memory allocation, just a view

Features: No allocation, efficient for read-only operations, prevents unnecessary copies

4. Raw String Literals (C++11)

Strings without escape sequence processing.

string path = R"(C:\Users\Name\file.txt)"; string json = R"({"key": "value"})";

Features: No escaping needed, multiline support, great for regex/JSON/paths

5. Wide Strings (wstring)

For Unicode and international character support.

std::wstring ws = L"Hello ‰∏ñÁïå"; // UTF-16 encoding on most platforms

Features: Unicode support, larger character size (2-4 bytes)

6. String Streams

String-based I/O operations and conversions.

#include <sstream> stringstream ss; ss << "Value: " << 123; string result = ss.str();

Features: String manipulation, type conversion, parsing

Common Mistakes & Pitfalls

‚ùå Buffer Overflow (C-style)

Most dangerous string error - writing beyond allocated memory.

// ‚ùå WRONG - Buffer overflow! char str[5]; strcpy(str, "Hello World"); // Overflow! // ‚úÖ CORRECT - Use std::string std::string str = "Hello World"; // Safe

‚ùå Not Null-Terminating

Forgetting '\0' causes undefined behavior.

// ‚ùå WRONG - No null terminator char str[6] = {'H','e','l','l','o'}; cout << str; // Undefined! // ‚úÖ CORRECT char str[6] = {'H','e','l','l','o','\0'};

‚ùå Modifying String Literals

String literals are read-only in memory.

// ‚ùå WRONG - Undefined behavior char* str = "Hello"; str[0] = 'h'; // CRASH! // ‚úÖ CORRECT - Use array char str[] = "Hello"; str[0] = 'h'; // OK

‚ùå Comparing with == (C-style)

== compares pointers, not content.

// ‚ùå WRONG - Compares addresses char s1[] = "Hello"; char s2[] = "Hello"; if (s1 == s2) // Always false! // ‚úÖ CORRECT - Use strcmp if (strcmp(s1, s2) == 0) // Correct

‚ùå Off-by-One in Loops

Using <= instead of < causes out-of-bounds access.

// ‚ùå WRONG for (int i = 0; i <= str.length(); i++) // Out of bounds! // ‚úÖ CORRECT for (int i = 0; i < str.length(); i++)

‚ùå Not Handling Empty Strings

Always check if string is empty before accessing.

// ‚ùå WRONG char first = str[0]; // Crash if empty! // ‚úÖ CORRECT if (!str.empty()) { char first = str[0]; }
üí° PRO TIP: Always use std::string for new code. It prevents most common string errors automatically!

Intermediate Operations

üîÑ Reverse String

Two-pointer technique - swap from both ends.

void reverseString(string& s) { int left = 0, right = s.length() - 1; while (left < right) { swap(s[left++], s[right--]); } } // O(n) time, O(1) space

üîÅ Check Palindrome

Compare characters from both ends moving inward.

bool isPalindrome(string s) { int left = 0, right = s.length() - 1; while (left < right) { if (s[left++] != s[right--]) return false; } return true; } // O(n) time

üîÄ Anagram Check

Two strings are anagrams if they have same character frequencies.

// Method 1: Sort bool isAnagram(string s1, string s2) { sort(s1.begin(), s1.end()); sort(s2.begin(), s2.end()); return s1 == s2; } // O(n log n) // Method 2: Hash map bool isAnagram(string s1, string s2) { if (s1.length() != s2.length()) return false; unordered_map<char, int> freq; for (char c : s1) freq[c]++; for (char c : s2) { if (--freq[c] < 0) return false; } return true; } // O(n)

üìä Character Frequency

Count occurrences of each character.

unordered_map<char, int> getFrequency(string s) { unordered_map<char, int> freq; for (char c : s) { freq[c]++; } return freq; } // O(n) time, O(k) space where k = unique chars

üî¢ String Compression

Run-length encoding: "aaabbc" ‚Üí "a3b2c1"

string compress(string s) { string result; int count = 1; for (int i = 1; i <= s.length(); i++) { if (i < s.length() && s[i] == s[i-1]) { count++; } else { result += s[i-1]; result += to_string(count); count = 1; } } return result.length() < s.length() ? result : s; }

‚úÇÔ∏è Remove Duplicates

Keep only first occurrence of each character.

string removeDuplicates(string s) { unordered_set<char> seen; string result; for (char c : s) { if (seen.find(c) == seen.end()) { result += c; seen.insert(c); } } return result; }

Common Techniques & Patterns

üéØ Two-Pointer Technique

When: Palindromes, reversing, removing characters, in-place modifications

Pattern: Start from both ends, move toward center or same direction

// Valid Palindrome (ignore non-alphanumeric) bool isPalindrome(string s) { int left = 0, right = s.length() - 1; while (left < right) { while (left < right && !isalnum(s[left])) left++; while (left < right && !isalnum(s[right])) right--; if (tolower(s[left++]) != tolower(s[right--])) return false; } return true; }

ü™ü Sliding Window

When: Substrings with constraints, longest/shortest substring problems

Pattern: Expand right, shrink left when condition violated

// Longest Substring Without Repeating Characters int lengthOfLongestSubstring(string s) { unordered_set<char> seen; int maxLen = 0, left = 0; for (int right = 0; right < s.length(); right++) { while (seen.count(s[right])) { seen.erase(s[left++]); } seen.insert(s[right]); maxLen = max(maxLen, right - left + 1); } return maxLen; } // O(n) time

üìä Hashing / Frequency Map

When: Anagrams, character counting, first unique character

Pattern: Store character frequencies in hash map or array

// Group Anagrams vector<vector<string>> groupAnagrams(vector<string>& strs) { unordered_map<string, vector<string>> groups; for (string s : strs) { string key = s; sort(key.begin(), key.end()); groups[key].push_back(s); } vector<vector<string>> result; for (auto& pair : groups) { result.push_back(pair.second); } return result; }

üîç Pattern Matching (KMP)

When: Finding substring in O(n+m) time

Pattern: Build LPS array for pattern, use it to avoid backtracking

// KMP Pattern Matching vector<int> computeLPS(string pattern) { int m = pattern.length(); vector<int> lps(m, 0); int len = 0, i = 1; while (i < m) { if (pattern[i] == pattern[len]) { lps[i++] = ++len; } else { if (len != 0) len = lps[len-1]; else lps[i++] = 0; } } return lps; }
üí° PRO TIP: Master these 4 patterns (Two-Pointer, Sliding Window, Hashing, KMP) to solve 90% of string problems!

String Algorithms

1. KMP (Knuth-Morris-Pratt) Algorithm

Time: O(n+m) | Space: O(m)

Linear time pattern matching using LPS (Longest Prefix Suffix) array to avoid backtracking.

// KMP Search int KMPSearch(string text, string pattern) { int n = text.length(), m = pattern.length(); vector<int> lps = computeLPS(pattern); int i = 0, j = 0; while (i < n) { if (text[i] == pattern[j]) { i++; j++; } if (j == m) return i - j; // Found else if (i < n && text[i] != pattern[j]) { if (j != 0) j = lps[j-1]; else i++; } } return -1; // Not found }

2. Rabin-Karp Algorithm

Time: O(n+m) average | Space: O(1)

Rolling hash for pattern matching. Excellent for multiple pattern search.

// Rabin-Karp with rolling hash int rabinKarp(string text, string pattern) { int n = text.length(), m = pattern.length(); int d = 256; // Number of characters int q = 101; // Prime for hash int h = 1, p = 0, t = 0; // Calculate h = d^(m-1) % q for (int i = 0; i < m-1; i++) h = (h * d) % q; // Calculate hash for pattern and first window for (int i = 0; i < m; i++) { p = (d * p + pattern[i]) % q; t = (d * t + text[i]) % q; } // Slide window for (int i = 0; i <= n - m; i++) { if (p == t) { // Hash match, verify if (text.substr(i, m) == pattern) return i; } if (i < n - m) { t = (d*(t - text[i]*h) + text[i+m]) % q; if (t < 0) t += q; } } return -1; }

3. Z-Algorithm

Time: O(n+m) | Space: O(n+m)

Builds Z-array for linear time pattern matching.

// Z-Algorithm for pattern matching vector<int> buildZArray(string s) { int n = s.length(); vector<int> z(n); int l = 0, r = 0; for (int i = 1; i < n; i++) { if (i > r) { l = r = i; while (r < n && s[r] == s[r-l]) r++; z[i] = r - l; r--; } else { int k = i - l; if (z[k] < r - i + 1) z[i] = z[k]; else { l = i; while (r < n && s[r] == s[r-l]) r++; z[i] = r - l; r--; } } } return z; }
