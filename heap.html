Types & Properties

Binary Heap (Max/Min)

Most common. Each node ‚â§ 2 children. O(log n) operations.

D-ary Heap

Each node has D children. Trade-off: smaller height, more comparisons per level.

Fibonacci Heap

Advanced: O(1) amortized insert/decrease-key. Used in graph algorithms.

04
Basic Operations

Insert (Push)

Add to end, heapify up

Time: O(log n)

Extract Max/Min

Replace root with last, heapify down

Time: O(log n)

Peek (Top)

Return root element

Time: O(1)

Heapify Down

Fix heap property downward

Time: O(log n)

Heapify Up

Fix heap property upward

Time: O(log n)

Build Heap

Convert array to heap

Time: O(n) - not O(n log n)!

// Heapify Down (Max Heap) void heapifyDown(int i) { int largest = i; int left = 2*i + 1, right = 2*i + 2; if (left < size && heap[left] > heap[largest]) largest = left; if (right < size && heap[right] > heap[largest]) largest = right; if (largest != i) { swap(heap[i], heap[largest]); heapifyDown(largest); } } // Build Heap - O(n) void buildHeap() { for (int i = size/2 - 1; i >= 0; i--) heapifyDown(i); }

Techniques & Problems

Common Patterns:

  • Two Heap (Median): Max heap (lower half) + Min heap (upper half)
  • K-way Merge: Min heap with (value, array_idx, elem_idx)
  • Top K: Min heap size K for K largest, Max heap for K smallest
  • Heap + HashMap: Frequency tracking with priority ordering

Summary & Decision Tree

Key Properties

  • Complete binary tree structure
  • Heap property: max (parent ‚â• children) or min (parent ‚â§ children)
  • O(1) access to max/min element at root
  • O(log n) insert and delete operations
  • O(n) build heap (not n log n!)
  • Array-based: space efficient, no pointers
  • NOT for searching arbitrary elements (O(n))
