What is a Stack?

A Stack is a linear data structure that follows the LIFO (Last In First Out) principle. Think of it like a stack of plates - you can only add or remove plates from the top. The last plate you place on the stack is the first one you'll remove.

Key Characteristics:

  • LIFO Structure - Last In First Out order
  • One-end access - Operations only at top
  • O(1) operations - Push, pop, and peek all constant time
  • No random access - Can only access top element
  • Dynamic or fixed size - Depends on implementation
  • Essential for recursion - Function call management

03
Types of Stacks

1. Fixed Size Stack (Array-based)

Pre-allocated array with maximum capacity.

int stack[MAX_SIZE]; int top = -1;

Pros: Fast access, cache-friendly | Cons: Fixed capacity, stack overflow possible

2. Dynamic Stack (Linked List)

Linked list of nodes, grows as needed.

struct Node { int data; Node* next; }; Node* top = nullptr;

Pros: No size limit (within memory) | Cons: Extra pointer memory, slower

3. Dynamic Array Stack (Vector-based)

Uses dynamic array that automatically resizes.

std::vector<int> stack;

Pros: Automatic resizing, STL benefits | Cons: Occasional reallocation cost

4. std::stack (STL Adapter)

Standard template library stack - most commonly used.

#include <stack> std::stack<int> st; st.push(10); st.pop(); int top = st.top();

Pros: Easy to use, well-tested | Container: Uses deque by default

5. Min Stack / Max Stack

Special stack that tracks minimum or maximum element in O(1).

// Tracks minimum at each level std::stack<int> mainStack; std::stack<int> minStack;

Use Case: Get min/max in constant time

6. Monotonic Stack

Elements maintained in sorted order (increasing or decreasing).

// Keep stack in increasing order while (!st.empty() && st.top() > current) { st.pop(); } st.push(current);

Use Case: Next greater/smaller element problems

Basic Operations

üì• Push (Insert)

Add element to top of stack

  • Time: O(1)
  • Check for overflow first
  • Increment top pointer
  • Add element at top
void push(int x) { if (top >= MAX-1) return; // Overflow arr[++top] = x; }

üì§ Pop (Remove)

Remove and return top element

  • Time: O(1)
  • Check for underflow first
  • Return top element
  • Decrement top pointer
int pop() { if (top < 0) return -1; // Underflow return arr[top--]; }

üëÄ Top / Peek

View top element without removing

  • Time: O(1)
  • Check if stack empty
  • Return top element
  • Don't modify stack
int peek() { if (top < 0) return -1; return arr[top]; }

‚ùì isEmpty

Check if stack has no elements

  • Time: O(1)
  • Compare top with -1
  • Return true if empty
  • Essential before pop/peek
bool isEmpty() { return top < 0; }

‚ö†Ô∏è isFull

Check if stack reached capacity

  • Time: O(1)
  • Only for array-based
  • Compare top with MAX-1
  • Prevents overflow
bool isFull() { return top >= MAX-1; }

üìè Size

Get number of elements in stack

  • Time: O(1)
  • Return top + 1
  • 0 if stack empty
  • Useful for iteration
int size() { return top + 1; }
üí° REMEMBER: Always check isEmpty() before pop() or top() to avoid underflow errors!

Common Mistakes & Pitfalls

‚ùå Stack Overflow (Push when full)

Most common error - pushing to a full stack causes overflow.

// ‚ùå WRONG - No overflow check void push(int x) { arr[++top] = x; // May overflow! } // ‚úÖ CORRECT - Check before push void push(int x) { if (isFull()) { throw std::overflow_error("Stack overflow!"); } arr[++top] = x; }

‚ùå Stack Underflow (Pop when empty)

Attempting to pop from an empty stack.

// ‚ùå WRONG - No underflow check int pop() { return arr[top--]; // May underflow! } // ‚úÖ CORRECT - Check before pop int pop() { if (isEmpty()) { throw std::underflow_error("Stack underflow!"); } return arr[top--]; }

‚ùå Not Checking Empty Before top()

Accessing top of empty stack causes crash.

// ‚ùå WRONG int val = st.top(); // Crash if empty! // ‚úÖ CORRECT if (!st.empty()) { int val = st.top(); }

‚ùå Trying to Iterate Over Stack Directly

std::stack doesn't support iteration.

// ‚ùå WRONG - Can't iterate for (auto x : st) { } // ERROR! // ‚úÖ CORRECT - Copy to temp std::stack<int> temp = st; while (!temp.empty()) { cout << temp.top() << " "; temp.pop(); }

‚ùå Forgetting to Pop After Top

Reading top but not removing can cause memory issues.

// ‚ùå WRONG - Memory leak with pointers st.push(new int(5)); int* val = st.top(); // Never popped - memory leaked! // ‚úÖ CORRECT int* val = st.top(); st.pop(); delete val; // Clean up

‚ùå Off-by-One with Top Index

Incorrect top initialization or updates.

// ‚ùå WRONG - Top starts at 0 int top = 0; // First push will skip index 0! // ‚úÖ CORRECT - Top starts at -1 int top = -1; // Empty stack

Intermediate Operations

üîÑ Reverse a Stack

Reverse all elements using recursion.

void insertAtBottom(stack<int>& st, int x) { if (st.empty()) { st.push(x); return; } int temp = st.top(); st.pop(); insertAtBottom(st, x); st.push(temp); } void reverse(stack<int>& st) { if (st.empty()) return; int temp = st.top(); st.pop(); reverse(st); insertAtBottom(st, temp); } // O(n¬≤) time

üìä Sort a Stack

Sort stack elements using recursion.

void sortedInsert(stack<int>& st, int x) { if (st.empty() || x > st.top()) { st.push(x); return; } int temp = st.top(); st.pop(); sortedInsert(st, x); st.push(temp); } void sortStack(stack<int>& st) { if (st.empty()) return; int temp = st.top(); st.pop(); sortStack(st); sortedInsert(st, temp); } // O(n¬≤) time

üéØ Get Middle Element

Find middle element using recursion.

int getMiddle(stack<int>& st, int curr) { if (curr == st.size() / 2) { return st.top(); } int temp = st.top(); st.pop(); int mid = getMiddle(st, curr + 1); st.push(temp); return mid; }

‚ùå Delete Middle Element

Remove middle element recursively.

void deleteMiddle(stack<int>& st, int curr, int mid) { if (curr == mid) { st.pop(); return; } int temp = st.top(); st.pop(); deleteMiddle(st, curr + 1, mid); st.push(temp); }

Common Techniques & Patterns

üîÅ Monotonic Stack

When: Next Greater/Smaller Element, Histogram problems

Pattern: Maintain stack in increasing/decreasing order

// Next Greater Element vector<int> nextGreater(vector<int>& arr) { stack<int> st; vector<int> result(arr.size(), -1); for (int i = arr.size() - 1; i >= 0; i--) { while (!st.empty() && st.top() <= arr[i]) { st.pop(); } if (!st.empty()) result[i] = st.top(); st.push(arr[i]); } return result; }

üìö Two Stack Technique

When: Min/Max Stack, Implement Queue using Stacks

// Min Stack - O(1) getMin() class MinStack { stack<int> mainStack; stack<int> minStack; public: void push(int x) { mainStack.push(x); if (minStack.empty() || x <= minStack.top()) { minStack.push(x); } } void pop() { if (mainStack.top() == minStack.top()) { minStack.pop(); } mainStack.pop(); } int getMin() { return minStack.top(); } };

üßÆ Expression Evaluation

When: Infix/Postfix evaluation, Calculator problems

// Evaluate Postfix Expression int evaluatePostfix(string exp) { stack<int> st; for (char c : exp) { if (isdigit(c)) { st.push(c - '0'); } else { int b = st.top(); st.pop(); int a = st.top(); st.pop(); if (c == '+') st.push(a + b); else if (c == '-') st.push(a - b); else if (c == '*') st.push(a * b); else if (c == '/') st.push(a / b); } } return st.top(); }

üîô Backtracking with Stack

When: DFS, maze solving, state exploration

// DFS using explicit stack void DFS(Graph g, int start) { stack<int> st; vector<bool> visited(g.V, false); st.push(start); while (!st.empty()) { int v = st.top(); st.pop(); if (!visited[v]) { visited[v] = true; // Process vertex for (int u : g.adj[v]) { if (!visited[u]) st.push(u); } } } }
üí° PATTERN TIP: Monotonic Stack + Two Stack + Expression Evaluation = 90% of stack interview problems!

Key Properties Summary

  • LIFO (Last In First Out) principle

  • Only top element accessible
  • No random access allowed
  • Can be array or linked list based
  • Essential for recursion & backtracking
  • Perfect for expression evaluation
  • Used in compilers, browsers, editors
