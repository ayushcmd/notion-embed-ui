What is Sorting?

Sorting is the process of arranging elements in a particular order (ascending or descending). It's one of the most fundamental operations in computer science.

Key Properties:

  • Stable: Equal elements maintain relative order (Merge, Insertion)
  • In-place: O(1) extra space (Bubble, Selection, Quick, Heap)
  • Adaptive: Faster on partially sorted data (Insertion, Bubble)
  • Comparison-based: Uses comparisons (most algorithms)
  • Non-comparison: Uses other properties (Counting, Radix)

  • Descending: Bade se chhote ‚Üí [90, 64, 34, 22, 11]

Kyun zaroori?

  • Searching fast (Binary Search)
  • Data organized
  • Duplicates easily find
  • Many algorithms ko sorted data chahiye

03
Bubble Sort

Algorithm

Compare adjacent elements and swap if in wrong order. Largest element "bubbles up" to the end each pass.

Time: O(n¬≤) Space: O(1) Stable: Yes
// Bubble Sort Implementation void bubbleSort(int arr[], int n) { for (int i = 0; i < n - 1; i++) { bool swapped = false; // Last i elements already sorted for (int j = 0; j < n - i - 1; j++) { if (arr[j] > arr[j + 1]) { swap(arr[j], arr[j + 1]); swapped = true; } } // If no swap, already sorted if (!swapped) break; } }

üáÆüá≥ BUBBLE SORT

Kaise kaam karta:

  • Do adjacent elements compare karo
  • Galat order ‚Üí Swap karo
  • Sabse bada element "bubble" ki tarah upar!

Pros: ‚úÖ Simple, ‚úÖ Adaptive (sorted pe O(n))

Cons: ‚ùå Bahut slow O(n¬≤)

Use: Teaching, tiny arrays (n < 10)

Insertion Sort

Algorithm

Build sorted array one element at a time. Insert each element at correct position in sorted part. Like sorting playing cards!

Time: O(n¬≤) Space: O(1) Stable: Yes Adaptive: Yes
// Insertion Sort Implementation void insertionSort(int arr[], int n) { for (int i = 1; i < n; i++) { int key = arr[i]; int j = i - 1; // Move elements > key one position ahead while (j >= 0 && arr[j] > key) { arr[j + 1] = arr[j]; j--; } arr[j + 1] = key; } }

üáÆüá≥ INSERTION SORT

Kaise kaam karta:

  • Ek-ek element uthao
  • Sorted part mein sahi jagah INSERT karo
  • Jaise playing cards arrange karte ho!

Pros: ‚úÖ Simple, ‚úÖ Best for small/nearly sorted, ‚úÖ Online

Cons: ‚ùå O(n¬≤) on random data

Use: Small arrays (n < 50), nearly sorted, hybrid sorts

Merge Sort (Divide & Conquer)

Algorithm

Divide array into halves, recursively sort, then merge sorted halves.

Time: O(n log n) Space: O(n) Stable: Yes
void merge(int arr[], int l, int m, int r) { int n1 = m - l + 1, n2 = r - m; int L[n1], R[n2]; for (int i = 0; i < n1; i++) L[i] = arr[l + i]; for (int j = 0; j < n2; j++) R[j] = arr[m + 1 + j]; int i = 0, j = 0, k = l; while (i < n1 && j < n2) { if (L[i] <= R[j]) arr[k++] = L[i++]; else arr[k++] = R[j++]; } while (i < n1) arr[k++] = L[i++]; while (j < n2) arr[k++] = R[j++]; } void mergeSort(int arr[], int l, int r) { if (l >= r) return; int m = l + (r - l) / 2; mergeSort(arr, l, m); // Sort left mergeSort(arr, m + 1, r); // Sort right merge(arr, l, m, r); // Merge }

üáÆüá≥ MERGE SORT

Divide and Conquer:

  • DIVIDE: Array ko 2 halves mein todo
  • CONQUER: Dono halves sort karo (recursively)
  • MERGE: Do sorted arrays merge karo

Pros: ‚úÖ Guaranteed O(n log n), ‚úÖ Stable, ‚úÖ Predictable

Cons: ‚ùå O(n) extra space, ‚ùå Not in-place

Use: Large data, stability chahiye, linked lists

06
Quick Sort (Divide & Conquer)

Algorithm

Choose pivot, partition array (smaller left, larger right), recursively sort partitions.

Avg: O(n log n) Worst: O(n¬≤) Space: O(log n) Stable: No
int partition(int arr[], int low, int high) { int pivot = arr[high]; int i = low - 1; for (int j = low; j < high; j++) { if (arr[j] < pivot) { i++; swap(arr[i], arr[j]); } } swap(arr[i + 1], arr[high]); return i + 1; } void quickSort(int arr[], int low, int high) { if (low < high) { int pi = partition(arr, low, high); quickSort(arr, low, pi - 1); quickSort(arr, pi + 1, high); } }

üáÆüá≥ QUICK SORT

Kaise kaam karta:

  • PIVOT: Ek element choose karo (usually last)
  • PARTITION: Chhote left, bade right
  • RECURSE: Dono sides sort karo

Pros: ‚úÖ Average O(n log n) - FAST!, ‚úÖ In-place, ‚úÖ Cache-friendly

Cons: ‚ùå Worst O(n¬≤) on sorted (fix: random pivot!)

Use: General purpose - BEST! C++ std::sort uses it

07
Heap Sort

Algorithm

Build max heap, repeatedly extract maximum (root) and heapify.

Time: O(n log n) Space: O(1) Stable: No
void heapify(int arr[], int n, int i) { int largest = i; int left = 2 * i + 1; int right = 2 * i + 2; if (left < n && arr[left] > arr[largest]) largest = left; if (right < n && arr[right] > arr[largest]) largest = right; if (largest != i) { swap(arr[i], arr[largest]); heapify(arr, n, largest); } } void heapSort(int arr[], int n) { // Build heap for (int i = n / 2 - 1; i >= 0; i--) heapify(arr, n, i); // Extract elements for (int i = n - 1; i > 0; i--) { swap(arr[0], arr[i]); heapify(arr, i, 0); } }

üáÆüá≥ HEAP SORT

Steps:

  • Max heap banao (parent > children)
  • Root (max) ko last se swap
  • Heap size kam karo, heapify
  • Repeat

Pros: ‚úÖ Guaranteed O(n log n), ‚úÖ In-place O(1)

Cons: ‚ùå Not stable, ‚ùå Slower than Quick in practice

Use: Memory constrained, real-time systems
