What is a Pointer?

A Pointer is a variable that stores the memory address of another variable. It "points to" the location in memory where data is stored, enabling direct memory access and manipulation.

Visual Representation:

p (Pointer)
0x7ffd1234
stores address
‚Üí
x (Variable)
42
at 0x7ffd1234

Pointer p contains the address where variable x is stored

// Pointer basics int x = 42; // Regular variable int* p = &x; // p points to x (&x = address of x) cout << x; // 42 (value of x) cout << &x; // 0x7ffd1234 (address of x) cout << p; // 0x7ffd1234 (value of p = address) cout << *p; // 42 (dereference = value at address) // Modify through pointer *p = 100; // x is now 100

Key Operators:

  • & = Address-of operator (get memory address)
  • * = Dereference operator (get value at address)
  • int* = Pointer to int type
  • nullptr = Null pointer (C++11, points to nothing)
  • NULL = Old-style null (defined as 0)

Common Mistakes & Pitfalls

‚ùå Dangling Pointer

// WRONG - Using freed memory int* p = new int(42); delete p; cout << *p; // UNDEFINED! p is dangling! // CORRECT - Set to nullptr after delete delete p; p = nullptr;

‚ùå Memory Leak

// WRONG - Never freed void func() { int* p = new int(42); return; // Memory leaked! } // CORRECT void func() { int* p = new int(42); delete p; // Free before return }

‚ùå Null Pointer Dereference

// WRONG - Crash! int* p = nullptr; *p = 42; // SEGMENTATION FAULT! // CORRECT - Check first if (p != nullptr) { *p = 42; }

‚ùå Double Delete

// WRONG int* p = new int(42); delete p; delete p; // UNDEFINED BEHAVIOR! // CORRECT delete p; p = nullptr; // Prevents double delete

‚ùå Wrong delete for Array

// WRONG int* arr = new int[10]; delete arr; // UNDEFINED BEHAVIOR! // CORRECT delete[] arr; // Use delete[] for arrays

‚ùå Returning Local Variable Address

// WRONG - x destroyed after return int* bad() { int x = 42; return &x; // DANGLING POINTER! } // CORRECT - Use dynamic memory int* good() { int* p = new int(42); return p; // OK - heap allocated }

Smart Pointers Deep Dive

unique_ptr - Sole Ownership

#include <memory> unique_ptr<int> p = make_unique<int>(42); *p; // Dereference p.get(); // Get raw pointer p.reset(); // Delete and set null // Transfer ownership (can't copy) unique_ptr<int> q = move(p); // p is now null

shared_ptr - Shared Ownership

shared_ptr<int> p1 = make_shared<int>(42); shared_ptr<int> p2 = p1; // Reference count = 2 p1.use_count(); // 2 p2.reset(); // Count goes to 1 p1.reset(); // Count goes to 0, memory freed

weak_ptr - Breaks Circular References

shared_ptr<int> sp = make_shared<int>(42); weak_ptr<int> wp = sp; // Must lock to use if (auto locked = wp.lock()) { *locked; // Safe to use }

Summary & Decision Tree

Key Properties

  • Pointer stores memory address of variable
  • & gets address, * dereferences (gets value)
  • nullptr for null/uninitialized pointers
  • Always delete dynamic memory to prevent leaks
  • Pointer arithmetic moves by sizeof(type)
  • Essential for linked structures (lists, trees)
  • Prefer smart pointers in modern C++
