What is a Linked List?

A Linked List is a dynamic linear data structure consisting of nodes where each node contains data and a pointer/reference to the next node in the sequence. Unlike arrays, linked lists are stored in non-contiguous memory locations on the heap.

Node Structure:

Data
Value
Next Pointer

struct Node {
    int data;
    Node* next;
};

Key Characteristics:

  • Dynamic memory allocation using new operator
  • Stored in heap memory (non-contiguous)
  • Each node points to the next node
  • No fixed size - grows/shrinks dynamically

Representation in Memory

Node Structure in C++:

class Node {
public:
    int data;       // Data stored in node
    Node* next;   // Pointer to next node
    
    Node(int val) {
        data = val;
        next = nullptr;
    }
};

Key Components:

Head Pointer

Points to the first node in the list. If head is NULL, the list is empty.

Tail Pointer (Optional)

Points to the last node for O(1) insertion at end.

NULL Termination

Last node's next pointer is NULL, marking the end of the list.

Heap Memory

Nodes are dynamically allocated using 'new' operator in C++.

Intermediate Operations

üîÑ Reverse Linked List

Iterative: Use three pointers (prev, curr, next) to reverse links
Recursive: Reverse sublist and adjust pointers recursively

‚ö° Find Middle Node

Use Fast & Slow Pointer technique: slow moves 1 step, fast moves 2 steps. When fast reaches end, slow is at middle.

üîÅ Detect Loop/Cycle

Floyd's Cycle Detection Algorithm: Use two pointers at different speeds. If they meet, a cycle exists.

‚ùå Remove Loop

After detecting loop, find the start of loop and break the cycle by setting appropriate pointer to NULL.

üîó Merge Two Sorted Lists

Compare nodes from both lists and build a new sorted list by adjusting pointers.

üìä Sort Linked List

Merge Sort is preferred for linked lists (O(n log n)) as it doesn't require random access.

STL in C++

Standard Template Library Support:

// Singly Linked List
#include <forward_list>
std::forward_list<int> flist = {1, 2, 3};
flist.push_front(0);  // Insert at beginning

// Doubly Linked List
#include <list>
std::list<int> dlist = {1, 2, 3};
dlist.push_back(4);   // Insert at end
dlist.push_front(0);  // Insert at beginning

std::forward_list

  • Singly linked list
  • More memory efficient
  • Forward iteration only
  • No size() method

std::list

  • Doubly linked list
  • Bidirectional iteration
  • Has size() method
  • More flexible operations

Common Operations

Iterator-Based

12
Key Properties Summary

Essential Takeaways

  • Dynamic size - grows and shrinks at runtime
  • Non-contiguous memory - nodes scattered in heap
  • Pointer-based structure - nodes connected via pointers
  • Linear traversal required - must start from head
  • No random access - cannot directly access by index
  • Efficient insertion/deletion at beginning - O(1)
  • Extra memory overhead - for storing pointers
  • Poor cache locality - compared to arrays

When to Use Linked Lists:

‚úì Frequent insertions/deletions at beginning
‚úì Unknown or dynamically changing size
‚úì No need for random access
‚úì Implementing other data structures (stacks, queues, graphs)
‚úì Memory fragmentation is acceptable

‚úó Avoid when: Need frequent random access, working with sorted data requiring binary search, memory overhead is critical, or cache performance is important
