What is a Tree?

A Tree is a hierarchical non-linear data structure with nodes connected by edges, representing parent-child relationships. It's fundamental for hierarchies like file systems and organizational charts.

Binary Tree Structure:

struct Node { int data; Node *left, *right; }; Node* createNode(int data) { Node* node = new Node(); node->data = data; node->left = node->right = nullptr; return node; }

Terminology:

  • Root: Top node (10)
  • Leaf: Node with no children (3,7,20)
  • Height: Longest path from node to leaf
  • Depth: Path from root to node

Types of Trees

Binary Tree

At most 2 children per node

Binary Search Tree (BST)

Left < Root < Right property. In-order gives sorted output.

AVL Tree

N-ary Tree

Each node can have N children

Trie (Prefix Tree)

String storage. Use: Autocomplete, spell check

03
Tree Traversals

Inorder (L-Root-R)

void inorder(Node* root) { if (!root) return; inorder(root->left); cout << root->data; inorder(root->right); }

BST: Gives sorted order

Preorder (Root-L-R)

void preorder(Node* root) { if (!root) return; cout << root->data; preorder(root->left); preorder(root->right); }

Use: Copy tree, serialize

Postorder (L-R-Root)

void postorder(Node* root) { if (!root) return; postorder(root->left); postorder(root->right); cout << root->data; }

Use: Delete tree, eval

Level Order (BFS)

queue<Node*> q; q.push(root); while (!q.empty()) { Node* n = q.front(); q.pop(); cout << n->data; if (n->left) q.push(n->left); if (n->right) q.push(n->right); }

Summary & Decision Tree

Key Properties

  • Hierarchical non-linear structure
  • One root, multiple leaves
  • No cycles (acyclic graph)
  • Unique path from root to any node
  • Recursive structure (subtrees)
  • O(log n) operations in balanced trees
  • O(n) operations in worst case (skewed)
