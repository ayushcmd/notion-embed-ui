What is a Graph?

A Graph is a non-linear data structure consisting of vertices (nodes) connected by edges. It represents relationships, networks, and connections. Mathematically: G = (V, E) where V is the set of vertices and E is the set of edges.

Key Terminology:

  • Vertex (Node): Point in the graph
  • Edge: Connection between two vertices
  • Adjacent: Vertices connected by an edge
  • Degree: Number of edges at a vertex
  • Path: Sequence of vertices connected by edges
  • Cycle: Path that starts and ends at same vertex
  • Connected: Path exists between all vertex pairs
  • Weighted: Edges have associated values

Graph Representations

1. Adjacency List

Array of vectors - each vertex has a list of neighbors.

vector<int> adjList[V]; // Add edge (undirected) adjList[u].push_back(v); adjList[v].push_back(u); // Weighted graph vector<pair<int,int>> adjList[V]; adjList[u].push_back({v, weight});

Pros: O(V+E) space, fast iteration | Cons: O(V) edge check

2. Adjacency Matrix

2D array - adjMatrix[i][j] = 1 if edge exists.

vector<vector<int>> adjMatrix(V, vector<int>(V, 0)); // Add edge adjMatrix[u][v] = 1; adjMatrix[v][u] = 1; // Undirected // Weighted adjMatrix[u][v] = weight;

Pros: O(1) edge check | Cons: O(V¬≤) space

3. Edge List

List of all edges - simple but limited.

struct Edge { int src, dest, weight; }; vector<Edge> edges;

Use: Kruskal's MST, simple storage

Shortest Path Algorithms

Dijkstra's Algorithm (Non-negative weights)

vector<int> dijkstra(int src, int V) { vector<int> dist(V, INT_MAX); priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq; dist[src] = 0; pq.push({0, src}); while (!pq.empty()) { auto [d, u] = pq.top(); pq.pop(); if (d > dist[u]) continue; for (auto [v, w] : adjList[u]) { if (dist[u] + w < dist[v]) { dist[v] = dist[u] + w; pq.push({dist[v], v}); } } } return dist; }

Time: O((V+E) log V) ‚Ä¢ Use: GPS, network routing

Bellman-Ford (Handles negative weights)

Relaxes all edges V-1 times. Can detect negative cycles.

Time: O(VE) ‚Ä¢ Use: Negative edge graphs

Floyd-Warshall (All-pairs shortest path)

Dynamic programming for all pairs. Time: O(V¬≥)

Minimum Spanning Tree

Kruskal's Algorithm

Sort edges, add if no cycle (Union-Find)

Time: O(E log E)

Best for: Sparse graphs

Prim's Algorithm

Grow tree from vertex (Min-Heap)

Time: O(E log V)

Best for: Dense graphs

üí° MST TIP: Both find minimum spanning tree. Kruskal better for sparse, Prim for dense graphs!

Summary & Decision Tree

Key Properties

  • Non-linear structure with vertices and edges
  • Many representations: adjacency list (sparse), matrix (dense)
  • Two core traversals: BFS (queue, shortest path) and DFS (stack, cycles)
  • Rich algorithms: Dijkstra, Bellman-Ford, MST, topological sort
  • Essential for networks, paths, relationships, dependencies
  • Most complex DS - requires multiple algorithms mastery

üéØ Graph Algorithm Decision Tree

Shortest path (weighted, non-negative)? ‚Üí Dijkstra
Shortest path (negative edges)? ‚Üí Bellman-Ford
All-pairs shortest path? ‚Üí Floyd-Warshall
Minimum spanning tree? ‚Üí Kruskal or Prim

Detect cycles? ‚Üí DFS (directed), Union-Find (undirected)

Grid traversal? ‚Üí DFS or BFS
